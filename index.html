Guía Rápida de MongoDB

Conceptos clave, diseño, gestión de BD, consultas (CRUD), agregación, seguridad y optimización de rendimiento.

1. Arquitectura

  • NoSQL Orientada a Documentos: Almacena datos en estructuras BSON (similar a JSON).
  • Equivalencias vs SQL:
    • Tabla = Colección
    • Registro = Documento
  • Atributo _id: Todo documento tiene un _id único que garantiza su unicidad.
  • Puerto por defecto: 27017.

2. Crear Base de Datos

En la consola de comandos de Mongo (mongosh):

use nombre_de_tu_bd
  • Creación Diferida: MongoDB no crea físicamente la base de datos ni la colección en el momento de ejecutar use. Se crearán en el disco duro solo hasta que insertes el primer documento.
  • Listar Bases de Datos: Utiliza el comando show dbs para ver las bases de datos existentes que ya contienen datos.

3. Patrones de Diseño

  • 1:1 Embebidos Un documento dentro de otro (Ej. Usuario y su perfil).
  • 1:N Embebidos Un array de documentos dentro del principal (Ej. Usuario y sus múltiples teléfonos).
  • 1:N Referidos Guarda el _id de otros documentos para evitar repetición de datos masivos.

4. Operaciones CRUD Completas

El ciclo de vida de los datos: Crear, Leer, Actualizar y Eliminar.

C - Create (Insertar)

Moderna: insertOne({..}) o insertMany([{..}]).

*Nota: Usa try-catch para controlar errores de _id duplicado.

R - Read (Leer)

db.col.find({filtro}, {proyec})
  • Filtros: $gt, $lt, $eq.
  • Lógicos: $or, $and.
  • Proyec: 1 (mostrar), 0 (ocultar).

U - Update (Actualizar)

updateOne() o updateMany().

  • Usa $set para modificar campos específicos.
  • upsert: true (inserta si no encuentra coincidencia).

D - Delete (Eliminar)

deleteOne({cond}) borra uno.

deleteMany({cond}) borra varios.

¡Peligro! deleteMany({}) vacía la colección.

5. Importar Datos (mongoimport)

Se ejecuta en la terminal del Sistema Operativo (cmd, powershell, bash), NO en mongosh.

Desde archivo JSON

mongoimport --db=tienda --collection=productos --file=datos.json --jsonArray

El modificador --jsonArray es crucial si tu archivo JSON contiene un array de múltiples objetos [{}, {}].

Desde archivo CSV

mongoimport --db=tienda --collection=usuarios --type=csv --headerline --file=usuarios.csv

El modificador --headerline indica que la primera fila del CSV contiene los nombres de los atributos/campos.

6. Exportar Datos (mongoexport)

También se ejecuta desde la terminal del Sistema Operativo para sacar backups o reportes.

Exportación Básica (JSON)

mongoexport --db=tienda --collection=usuarios --out=usuarios.json --jsonArray

Saca toda la colección a un archivo. --out define el nombre del archivo de salida.

Exportación Condicional / CSV

mongoexport --db=tienda --collection=ventas --query='{"total": {"$gt": 100}}' --out=altas.json

Exportar con Filtro: Usa --query para exportar solo documentos que cumplan una condición específica.
Para CSV: Añade --type=csv y define las columnas obligatoriamente con --fields=nombre,edad.

7. Casos de Uso y Ejemplos Prácticos

Ejemplos aplicados en código simulando una base de datos de una "biblioteca".

1. Crear BD e Insertar (Create)

Ingresamos a la BD e insertamos un libro (se crea automáticamente).

// mongosh
use biblioteca
db.libros.insertOne({ titulo: "1984", autor: "George Orwell", año: 1949 })

2. Buscar (Read)

Buscamos libros que se publicaron antes del año 2000 usando $lt (less than).

// mongosh
db.libros.find({ año: { $lt: 2000 } })

3. Modificar (Update)

Le agregamos el nuevo campo "genero" al libro con título "1984".

// mongosh
db.libros.updateOne({ titulo: "1984" }, { $set: { genero: "Distopía" } })

4. Eliminar y Exportar

Eliminamos el registro en Mongo, y luego exportamos desde la terminal de nuestra PC.

// mongosh
db.libros.deleteOne({ titulo: "1984" }) # Terminal del sistema (cmd/bash)
mongoexport --db=biblioteca --collection=libros --out=libros_backup.json

9. Teorema CAP

Concepto clave (Mencionado en videoclase) para entender BD distribuidas como Mongo:

  • C - Consistencia: Los nodos ven la misma información al mismo tiempo.
  • A - Disponibilidad: El sistema funciona aunque falle una parte del clúster.
  • P - Tolerancia a Particiones: Opera aunque se pierda conexión de red entre nodos.

11. Agregación en MongoDB (Tema 4)

Técnicas para analizar, procesar y resumir grandes volúmenes de datos.

Deprecado

Método 1: MapReduce

Modelo clásico de Big Data. Filtra y agrupa datos emitiendo pares clave/valor.

  • ¡Atención! Prácticamente obsoleto en versiones recientes debido a su alta complejidad de programación. Mongo ya no apuesta por él.
  • Fase MAP: Emite los pares (Ej. clave constante "suma" y atributo "precio" como valor).
  • Fase REDUCE: Toma los pares agrupados y realiza la operación final (suma total).

Parámetros Extendidos:

  • query: Filtra antes de procesar.
  • sort: Orden previo.
  • limit: Límite de documentos.
  • finalize: Refina el resultado.
  • scope: Variables globales.
  • verbose: Tiempos de proceso.
db.productos.mapReduce( mapFn, reduceFn, { out: "resultados", query: { categoria: "A" } } )

Método 2: Aggregate (La Apuesta Actual)

Destacado por su relevancia y flexibilidad. Es tan potente que a menudo reemplaza al comando find() tradicional para búsquedas avanzadas.

  • $match: Funciona exactamente igual que los criterios de búsqueda de un find().
  • $group: Agrupa por _id y aplica transformaciones con operadores muy especializados.
  • $sort: Retorna la salida en un orden específico.
// Ejemplo: Sumar ventas por región db.ventas.aggregate([ { $match: { estado: "Aprobado" } }, { $group: { _id: "$region", total: { $sum: "$monto" } } } ])

Profundización: Etapas Avanzadas del Pipeline

El Aggregation Framework se vuelve extremadamente potente cuando combinamos múltiples etapas. A continuación, las operaciones más críticas para cruzar y transformar datos:

$lookup El "JOIN" de Mongo

Cruza datos de dos colecciones distintas. Une documentos de una colección externa basándose en un campo local y uno foráneo. Genera un nuevo array con los datos cruzados.

$unwind Desenrollar Arrays

Si un documento tiene un array de 3 elementos, $unwind lo descompone y genera 3 documentos independientes. Es vital usarlo justo después de un $lookup.

$sort & $limit Orden y Paginación

$sort ordena los resultados pasados (1 ascendente, -1 descendente). $limit restringe la cantidad de documentos que continúan, ideal para un "Top 10".

$project Redimensionar

Formatea los documentos. Oculta campos innecesarios (0), muestra los requeridos (1), o crea campos calculados para la salida final.

$group Agrupación y Acumuladores

Agrupa por un _id definido y aplica cálculos matemáticos a todo el grupo de forma simultánea:

  • $sum: Sumatoria.
  • $avg: Promedio numérico.
  • $min / $max: Valor mínimo/máximo.
  • $first / $last: Extrae el primer/último valor.
// EJEMPLO MAESTRO: Top 3 Países con mayores compras, cruzando Ventas con Usuarios db.ventas.aggregate([ // 1. JOIN: Unimos la venta con la colección "usuarios" (por usuario_id) { $lookup: { from: "usuarios", localField: "usuario_id", foreignField: "_id", as: "datos_usuario" } }, // 2. DESENROLLAR: Convertimos el array "datos_usuario" en documentos planos { $unwind: "$datos_usuario" }, // 3. FILTRAR: Solo procesar ventas que sí se concretaron { $match: { estado: "Finalizado" } }, // 4. AGRUPAR: Por país del usuario y sumar todos los montos de compra { $group: { _id: "$datos_usuario.pais", total_ventas: { $sum: "$monto" } } }, // 5. ORDENAR: De mayor a menor total de ventas (-1) { $sort: { total_ventas: -1 } }, // 6. LIMITAR: Mostrar únicamente el Top 3 { $limit: 3 } ])

12. Creación de Índices y Rendimiento (Tema 5)

Evita el Collection Scan optimizando el tiempo de ejecución de las consultas en grandes colecciones.

Fundamento y Sintaxis

Hacer un find() sin índices obliga a Mongo a leer cada documento de la colección para encontrar lo que buscas. Para evitar esto, se usa createIndex(), el cual recibe dos argumentos JSON: las claves y las opciones.

// Sintaxis General
db.coleccion.createIndex(
  { claves }, // 1 para ascendente, -1 para descendente
  { opciones }
)

Catálogo de Opciones

Permiten personalizar drásticamente el comportamiento del índice:

  • background: Si es true, construye el índice en 2do plano sin bloquear otras operaciones de la BD.
  • unique: Si es true, rechaza insertar documentos si ya existe ese valor (Ej. un email duplicado).
  • name: Permite asignar una cadena de texto personalizada como nombre del índice.
  • sparse: Si es true, solo indexa los documentos que sí contengan ese atributo (ahorra espacio).
  • dropDups: Histórico/Deprecado. Eliminaba duplicados forzosamente al crear un índice único.
  • v: Versión del formato del índice (usado por el motor interno).

Ejemplo Práctico Aplicando Opciones

Supongamos que queremos indexar la colección "usuarios" por el campo "email". Queremos que la búsqueda sea rápida (ascendente), que ningún usuario pueda repetir email (unique), que solo indexe usuarios que sí registraron email (sparse) y que no detenga el servidor mientras se crea (background):

db.usuarios.createIndex(
  // 1. El objeto CLAVES
  { "email": 1 },
  // 2. El objeto OPCIONES
  {
    unique: true,
    background: true,
    sparse: true,
    name: "idx_email_unico"
  }
)

13. Ecosistema: Docker, Nube y Drivers

Del entorno local a la nube y cómo conectarlo a tus aplicaciones (Flujo del Mundo Real).

🐳 1. Entorno Local (Docker)

Evita instalar Mongo en tu sistema operativo, levántalo de forma aislada en un contenedor. El puerto local estándar es 27017.

# -d: Corre en segundo plano
# -p: Mapea tu puerto local con el del contenedor
# --name: Le da un nombre a la instancia
docker run -d -p 27017:27017 --name mi_mongo mongo:latest

# -> URI de Conexión Local:
mongodb://localhost:27017

☁️ 2. Entorno Cloud (Atlas)

Para producción, usa MongoDB Atlas (DBaaS). Te entregará una cadena de conexión (URI) que incluye credenciales, el clúster y la BD.

// Desglose de la Cadena de Conexión
mongodb+srv://admin:P@ssw0rd123@cluster0.abcde.mongodb.net/miBaseDatos?retryWrites=true

*Así se estructura la URI para que programas externos o MongoDB Compass se conecten.

🔌 3. Conexión a App (Drivers)

Tus programas (Python, Node.js, Java) usan librerías llamadas Drivers para enviar los comandos CRUD a la base de datos.

import pymongo

# 1. Establecer conexión (Usa tu URI de Atlas o Local)
cliente = pymongo.MongoClient("mongodb+srv://...")
# Apuntar a la colección "usuarios"
coleccion = cliente.miBaseDatos.usuarios

# 2. Flujo Completo: Insertar y Buscar
coleccion.insert_one({"rol": "Admin"})
resultado = coleccion.find_one({"rol": "Admin"})
print(resultado)

14. Gestión Avanzada, Seguridad y Sharding (Tema 5)

Administración de clústeres, roles de usuario y escalabilidad horizontal.

🔐 Seguridad Nativa y Cifrado

Gestión granular de accesos. Desde Mongo 3.2, incluye cifrado en reposo nativo (WiredTiger). Esto asegura la protección de los datos a nivel de almacenamiento físico del disco.

// Crear un administrador de BD
use admin
db.createUser({
  user: "adminSuper",
  pwd: "Pass123!",
  roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
})

🗄️ Sharding (Escalabilidad)

Permite repartir la carga de datos masivos entre diferentes servidores (nodos) para garantizar la escalabilidad futura y optimizar el rendimiento. Requiere una Clave de Fragmentación (Shard Key).

// 1. Habilitar Sharding en la BD
sh.enableSharding("tienda")

// 2. Fragmentar colección por una clave
sh.shardCollection("tienda.ventas", { "pais_origen": 1 })

🎭 Catálogo de Roles Predefinidos

MongoDB incluye roles con niveles de acceso granulares para mantener la seguridad en producción y limitar privilegios:

dbOwner

Dueño de la Base de Datos. Otorga control total sobre una BD específica. Combina permisos de lectura, escritura y la capacidad de administrar a otros usuarios (dbAdmin y userAdmin).

clusterMonitor

Solo lectura. Ideal para herramientas de supervisión. Permite acceder a métricas y estadísticas del sistema sin poder visualizar la información de los usuarios ni modificar configuraciones.

hostManager

Admin de Servidores. Permite gestionar servicios individuales (mongod o mongos), cambiar configuraciones del servidor en tiempo real, forzar cierres (shutdown) y gestionar registros (logs).

clusterAdmin / Manager

Alta Administración. clusterManager gestiona replicación y fragmentación (Sharding). clusterAdmin es el rango máximo a nivel de la arquitectura: une todos los roles de clúster.

15. Protección en Producción (Límites y Restricciones)

Configuraciones críticas en el archivo mongod.conf para evitar sobrecargas, ataques y restringir el acceso a herramientas.

🛑 Limitar Conexiones

Por defecto, Mongo permite miles de conexiones simultáneas. En producción, un error en tu código o un ataque puede agotar la memoria RAM del servidor rápidamente. Es vital establecer un techo con el parámetro maxIncomingConnections.

🚧 Restricción de Red e IPs

Nunca expongas Mongo a todo internet (IP 0.0.0.0). Usa bindIp para limitar la escucha exclusivamente a la IP interna de tu aplicación. Esto bloquea automáticamente herramientas externas no autorizadas (como un Compass ajeno) a nivel de red.

Ejemplo Real (mongod.conf)

# 1. Restricción de Red y Conexiones
net:
  port: 27017
  # Solo admite Localhost y la IP de tu App
  bindIp: 127.0.0.1, 10.0.0.15
  # Limita a 100 conexiones simultáneas
  maxIncomingConnections: 100

# 2. Obligar Autenticación a Herramientas
security:
  authorization: "enabled"

16. Estrategias de Respaldo y Migración (Tema 5)

Métodos seguros para respaldar bases de datos masivas: desde copias físicas hasta herramientas lógicas.

📁 1. Copia del Sistema de Ficheros (Física)

Consiste en copiar literalmente la carpeta donde MongoDB guarda los datos en el disco duro para moverla a otro servidor.

¡PRECAUCIÓN CRÍTICA!

Es obligatorio detener el servidor (mongod) antes de hacer la copia. Si el servidor está en ejecución, los datos se corromperán irremediablemente.

# 1. Apagar el servicio (Linux)
sudo systemctl stop mongod

# 2. Copiar el directorio de datos
cp -R /var/lib/mongo /backups/mongo_backup_fisico

# 3. Reiniciar el servicio
sudo systemctl start mongod

🛠️ 2. Herramientas Lógicas

mongodump y mongorestore permiten respaldar bases de datos en caliente (sin apagar el servidor). Son flexibles para servidores completos, BD individuales o colecciones específicas.

# mongodump: Respaldo de una BD específica
mongodump --db=tienda --out=/backups/logicos/

# mongorestore: Restaurar con puerto específico
mongorestore --port=27017 /backups/logicos/tienda

# mongorestore: Escenario complejo (con Auth)
mongorestore --host=10.0.0.15 --port=27017 \
  --username=admin --password=claveSegura \
  --authenticationDatabase=admin /backups/datos

⚖️ Replicación vs Backups (Aclaración Clave)

La videoclase hace una distinción vital para entornos de producción:

  • Backups (mongodump / Copia física): Son "instantáneas" (snapshots) en un momento específico del tiempo. Sirven para recuperar datos borrados accidentalmente o registros históricos.
  • Replicación de Nodos (Replica Sets): Es una copia continua y en tiempo real en múltiples servidores. Su objetivo no es recuperar datos borrados (porque el borrado también se replica instantáneamente), sino garantizar alta disponibilidad si un servidor se cae.

17. Escalabilidad Horizontal: Sharding (Tema 5)

Arquitectura para distribuir datos masivos a través de múltiples servidores, maximizando el rendimiento y el almacenamiento.

Los 3 Componentes Principales

Un Sharded Cluster no es un solo servidor; es una arquitectura completa formada por tres piezas clave que trabajan en sincronía:

📦 1. Shards

Contienen los subconjuntos reales de datos. En un entorno de producción, cada Shard suele ser un Replica Set para garantizar que no se pierdan datos si un servidor físico se apaga.

🚦 2. Query Routers (mongos)

Actúan como "directores de tráfico". Reciben las consultas de las aplicaciones clientes y saben exactamente a qué Shard enviarlas. La app nunca interactúa con el Shard directamente.

🧠 3. Config Servers

Almacenan los metadatos y la configuración del clúster. Son el "mapa" que le dice a los Query Routers en qué Shard se encuentra almacenado cada fragmento específico de información.

Pasos para Configurar un Sharded Cluster

Secuencia estricta para levantar esta arquitectura (vista en clase):

# PASO 1: Iniciar los Config Servers
# (Utilizan el puerto por defecto 27019)
mongod --configsvr --replSet confRepl --port 27019

# PASO 2: Iniciar los Query Routers (instancias mongos)
# Se le debe indicar a qué Config Server conectarse
mongos --configdb confRepl/localhost:27019 --port 27017

# PASO 3: Agregar Shards al clúster
# (Entramos a la terminal del 'mongos' y ejecutamos:)
sh.addShard("shard01/localhost:27018")

# PASO 4: Habilitar Sharding a nivel BD y Colección
# Habilitamos la base de datos:
sh.enableSharding("tienda")
# Fragmentamos la colección indicando el "Patrón" (Shard Key):
sh.shardCollection("tienda.ventas", { "pais_origen": 1 })

18. Integración con Aplicaciones (Drivers) - Tema 6

Cómo conectar y comunicar tus programas y el ecosistema Big Data con la base de datos.

🔌 ¿Qué es un Driver?

Es un software que actúa como intermediario entre la base de datos y el código de tu aplicación. Permite consultar y gestionar los datos sin importar el lenguaje en el que estés programando.

Lenguajes y Soporte Oficial

El ecosistema de MongoDB proporciona una amplia gama de drivers oficiales con soporte continuo para lenguajes populares como: Java, Python (PyMongo), Node.js, C#, Ruby, entre otros.

🔗 Comunidad y Ecosistema

En el análisis de datos masivos (Big Data), Mongo se integra con otras herramientas potentes mediante la comunidad y conectores especializados.

MongoDB Hadoop Connector

Es un plugin esencial que permite utilizar MongoDB como una fuente de entrada (input) o un destino de salida (output) en sistemas de procesamiento masivo basados en el framework de Hadoop.

☕ Ejemplo Práctico (Java)

El primer paso en Java es tener las librerías necesarias para el Driver. Hay dos opciones comunes:

  • Usar gestores como Maven (dependencias en pom.xml).
  • Descargar directamente los archivos .jar oficiales.
// 1. Conexión por defecto (localhost)
MongoClient cliente = new MongoClient();

// 2. Conexión a host y puerto
MongoClient cliente = new MongoClient("localhost", 27017);

// 3. Seleccionar la base de datos
MongoDatabase db = cliente.getDatabase("miApp");

🐍 Ejemplo Práctico (Python)

Python usa PyMongo, el driver preferido en entornos de Data Science y Analítica de Datos:

  • Instalación rápida vía terminal: pip install pymongo
# 1. Importar la librería
from pymongo import MongoClient

# 2. Conexión al servidor (Local o URI de Atlas)
cliente = MongoClient("mongodb://localhost:27017/")

# 3. Acceder a BD y Colección
db = cliente["miApp"]
coleccion = db["usuarios"]

# 4. Consultar un dato
resultado = coleccion.find_one({"rol": "Admin"})
print(resultado)
