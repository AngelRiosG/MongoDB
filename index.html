Guía Rápida de MongoDB

Conceptos clave, diseño, consultas y exportación de datos para la gestión de bases de datos masivas NoSQL.

1. Arquitectura

  • NoSQL Orientada a Documentos: Almacena datos en estructuras BSON (similar a JSON).
  • Equivalencias:
    • Tabla (SQL) = Collection
    • Registro (SQL) = Documento
  • Atributo _id: Todo documento tiene un _id único que garantiza su unicidad.
  • Puerto por defecto: 27017.

2. Patrones de Diseño

La forma de modelar los datos es vital para el rendimiento:

  • 1:1 Documentos Embebidos Un documento dentro de otro. Ej. Un usuario con su dirección incrustada.
  • 1:N Documentos Embebidos Un array de documentos dentro de uno principal. Ej. Un usuario y sus múltiples direcciones.
  • 1:N Documentos Referidos Para evitar repetición. Se guarda el _id de otros documentos (como llaves foráneas).

3. Consultas (Find)

Lectura de datos en colecciones:

db.coleccion.find({criterios}, {proyeccion})
  • Criterios: Condiciones de búsqueda usando operadores.
  • Proyección: Indica qué atributos mostrar u ocultar.
  • Operadores Lógicos: $or, $not.
  • Operadores de Comparación:
    $gt (mayor que), $in (incluido en).

4. Insertar y Actualizar

  • Insertar:

    db.coleccion.insert() o save(). Para inserciones múltiples se envían arrays. *Es clave usar try-catch para manejo de errores.

  • Actualizar:

    db.coleccion.update()

    • multi: true Modifica todos los documentos que coincidan.
    • upsert: true Inserta un documento nuevo si ninguno cumple la condición de búsqueda.
  • Eliminar:

    remove() o deleteOne() para borrar uno o varios registros.

5. Pasos para Exportar Datos (mongoexport)

La herramienta mongoexport permite exportar colecciones enteras o resultados de consultas a archivos JSON o CSV. Se ejecuta desde la terminal del sistema operativo, no desde la consola de MongoDB (mongosh).

Exportar a formato JSON

mongoexport --db=miBaseDatos --collection=usuarios --out=usuarios.json
  1. Abre tu terminal (cmd, bash, terminal).
  2. Indica la base de datos con --db.
  3. Indica la colección con --collection.
  4. Define el nombre de salida con --out.

Exportar a formato CSV

mongoexport --db=miBaseDatos --collection=usuarios --type=csv --fields=nombre,edad,email --out=usuarios.csv
  1. Añade la bandera --type=csv.
  2. Es obligatorio especificar los campos a exportar usando --fields separados por comas.
